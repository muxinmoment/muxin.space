# muxin.space 项目详细介绍指南

## 1. 项目定位

`muxin.space` 是一个基于 `Astro` 搭建的个人网站，定位为学习笔记、项目分享、个人主页和前端练手场。

这个站不是业务系统，也不是复杂后台，而是一个偏内容型、长期迭代的个人空间。它的核心目标有四个：

- 记录学习过程，不只记录结果
- 沉淀可复用的技术笔记
- 展示正在推进的项目
- 顺手练习前端页面、组件和站点结构设计

## 2. 为什么选 Astro

这个项目选择 `Astro`，不是为了追新，而是为了更贴合站点类型。

- 内容站优先：Astro 天然适合博客、笔记、文档、作品集
- 默认静态输出：页面轻，首屏好，部署简单
- Markdown 友好：适合直接把学习记录写成文章
- 渐进增强：以后需要交互时，可以按需加入局部 React 组件

如果这个站未来变成更强交互的网站，再考虑更重的应用框架会更合适；但在当前阶段，Astro 是性价比最高的选择。

## 3. 当前功能构成

站点目前包含这些页面和能力：

- 首页 `/
- 笔记列表 `/notes`
- 笔记详情 `/notes/[slug]`
- 项目页 `/projects`
- 关于页 `/about`
- 404 页面 `/404`

内容部分已经接入 `Astro Content Collections`，可以直接在 `src/content/notes/` 里新增 Markdown 笔记。

## 4. 技术栈

- 框架：`Astro`
- 语言：`TypeScript`
- 内容系统：`Markdown + Content Collections`
- 样式：`CSS` 单文件全局样式
- 部署目标：`Cloudflare Pages`

当前项目的设计原则是：先把静态站做扎实，再考虑引入更多动态能力。

## 6. 内容模型

当前只有一个内容集合：`notes`

笔记支持的字段如下：

- `title`：标题
- `description`：摘要
- `pubDate`：发布日期
- `updatedDate`：可选，更新日期
- `tags`：标签数组
- `draft`：草稿标记

笔记文件放在 `src/content/notes/`，比如：

```md
---
title: "为什么这个个人站先选 Astro"
description: "从内容站、学习成本和部署路径三个维度做的选型记录。"
pubDate: 2026-05-31
tags:
  - astro
  - frontend
draft: false
---

正文内容写在这里。
```

## 7. 页面说明

### 首页

首页负责做三件事：

- 给出站点定位
- 展示最近更新的笔记
- 展示当前项目片段和关注点

### 笔记页

`/notes` 是文章归档页，按时间排序展示所有非草稿笔记。

`/notes/[slug]` 是详情页，会渲染单篇 Markdown 内容。

### 项目页

`/projects` 用来展示你现在在推进的项目，当前主要是：

- Aido
- muxin.space
- Kacha.ai

### 关于页

`/about` 用来放个人说明、站点理念和长期目标。

## 8. 视觉风格

当前站点的视觉方向不是极简白板风，也不是默认模板风，而是偏温暖、有层次、适合个人内容沉淀的风格。

- 背景：渐变 + 纹理感
- 字体：正文与标题分层
- 卡片：圆角、阴影、轻边框
- 交互：按钮和导航状态有明确反馈
- 响应式：移动端会自动折叠为单列

这些设计的目标是让站点看起来更像“认真在经营的个人空间”，而不是一个临时拼出来的演示页。

## 10. Cloudflare Pages 部署

当前项目是纯静态站，最适合直接部署到 `Cloudflare Pages`。

### 推荐配置

- Framework preset: `Astro`
- Build command: `npm run build`
- Build output directory: `dist`
- Node version: `20` 或 `22`

### 部署流程

1. 把仓库推到 GitHub
2. 在 Cloudflare Pages 里连接这个仓库
3. 选择 `Astro` 预设
4. 检查构建命令和输出目录
5. 保存后自动触发首次部署

## 11. 如何新增一篇笔记

1. 在 `src/content/notes/` 下新建一个 `.md` 文件
2. 填写 frontmatter
3. 写正文内容
4. 保存后访问 `/notes`
5. 本地执行 `npm run build` 验证

## 12. 如何新增一个项目卡片

项目卡片数据放在 `src/data/site.ts`。

你只需要在 `projects` 数组里新增一个对象，包含：

- `name`
- `status`
- `summary`
- `stack`
- `highlights`

首页和项目页会自动读取这些数据。

## 14. 常见问题

### 构建失败怎么办

优先检查这三项：

- `npm install` 是否成功
- `npm run build` 是否在本地通过
- `src/content.config.ts` 和笔记 frontmatter 是否写对

### 为什么首页一开始只有一篇或几篇笔记

因为首页默认只展示最近的几篇内容，这是为了让首屏更聚焦。

### 为什么不用 React 作为主框架

因为这个站目前是内容站，不是复杂交互应用。Astro 更轻、更直观，也更适合学习型博客。

## 15. 一句话总结

`muxin.space` 的目标不是一次做完，而是变成一个能长期更新、能持续沉淀、能顺手练习前端的个人网站。
